# Staff-management

نظام متكامل لإدارة الموظفين وفرق العمل — مبني بـ **Next.js 16** و **Supabase**،
عربي بالكامل (RTL) مع دعم الإنجليزية.

> - الخطة الكاملة (المعمارية، الـ schema، المراحل): [`docs/ROADMAP.md`](./docs/ROADMAP.md)
> - دليل الاستخدام بالعربي: [`docs/USER_GUIDE_AR.md`](./docs/USER_GUIDE_AR.md)
> - دليل النشر على cPanel/Node: [`docs/DEPLOY_CPANEL_AR.md`](./docs/DEPLOY_CPANEL_AR.md)

## التقنيات

- **Next.js 16** (App Router, Server Actions) + **TypeScript** + **React 19**
- **Tailwind CSS v4** + **shadcn/ui** (RTL / عربي)
- **Supabase** (Auth + Postgres + RLS + Storage + Realtime)
- **next-intl** (عربي افتراضي، مع إنجليزي)
- **TanStack Query**، **React Hook Form** + **Zod**
- **Resend** (إيميل المعاملات والدعوات)
- **Google Drive API** (تكامل ملفات المشاريع)
- **Claude / Anthropic** (مساعدات الذكاء الاصطناعي)
- **Web Push** (VAPID) + **PWA** (يعمل offline)

## المميزات الكاملة

### 🔐 المصادقة والصلاحيات
- **بدون تسجيل عام** — الحسابات بيُنشئها المدير عبر دعوات بالإيميل.
- إعادة تعيين كلمة السر + استعادة عبر إيميل.
- **٤ أدوار:** مدير عام (`super_admin`)، قائد فريق (`team_leader`)، عضو فريق
  (`team_member`)، وموارد بشرية (`hr`).
- **HR** يدير بيانات الموظفين على مستوى الشركة كاملة (موظفين، أجازات، حضور،
  تقييمات، مؤشرات، بونص) — بدون صلاحيات الهيكل أو سجل التغييرات.
- **Row Level Security (RLS)** كامل على مستوى الداتابيز — كل دور يشوف ويعدّل
  ما يخصّه فقط، والصلاحيات مفروضة في Postgres نفسه مش في الواجهة بس.

### 🏢 الأقسام والمشاريع والفرق
- إدارة الأقسام مع قائد لكل قسم.
- مشاريع لكل قسم بحالات: تخطيط / نشط / متوقف مؤقتاً / مكتمل / ملغي.
- أعضاء المشروع بأدوار: قائد / عضو / مراقب.
- **أدوات القسم** — دليل أدوات القسم وبيانات الدخول الخاصة بها لأعضائه.

### 📝 التاسكات
- عرض **Kanban** (سحب وإفلات) أو **قائمة**.
- حالات: للعمل / قيد التنفيذ / مراجعة / مكتمل / ملغي.
- أولويات: منخفضة / متوسطة / عالية / عاجلة + مواعيد تسليم.
- **مهام فرعية** (subtasks) و**اعتماديات** بين التاسكات (dependencies).
- **تعليقات** مع **منشن** (@mention) ومرفقات ملفات.
- **تحديثات لحظية (Realtime)** للوحة الكانبان والتعليقات.
- **تشيك ليست** لكل مشروع + **قوالب تشيك ليست** قابلة لإعادة الاستخدام.

### 📁 الملفات وتكامل Google Drive
- ملفات لكل مشروع (تخزين Supabase Storage + مرفقات التاسكات).
- **فولدر تلقائي لكل مشروع** على Google Drive + **مزامنة صلاحيات** الأعضاء.
- روابط تحميل آمنة (signed URLs) دون كشف الـ buckets.

### 🔔 الإشعارات
- إشعارات **داخل التطبيق** + **إيميل** + **Web Push** (للموبايل عبر PWA).
- **تفضيلات لكل مستخدم** تتحكم في أنواع الإشعارات.
- جرس إشعارات بتحديث لحظي.

### ☀️ الستاندب اليومي وموازن الحمل
- ستاندب يومي: شغل أمس، خطة اليوم، العوائق، والمزاج.
- **ملخص ذكي (AI)** لستاندب الفريق يبرز الأولويات والعوائق.
- **بوت تذكير يومي** بالإيميل لمن لم يسجّل ستاندب.
- **موازن حمل العمل (Workload)** بنسبة إشغال كل عضو + **اقتراحات إعادة توزيع**.

### 🎯 المؤشرات والتقييمات
- تسجيل قيم **مؤشرات الأداء (KPIs)** للموظفين عبر فترات.
- **تقييمات أسبوعية / شهرية** مبنية على المؤشرات بأوزان ودرجة موزونة.
- حالات التقييم: مسودة / نهائي / مُرسَل.
- **تقارير PDF** جاهزة للطباعة لكل تقييم.
- **توليد التقييمات الشهرية تلقائياً** (cron).

### 🤝 تقييم الزملاء (360°)
- تقييم سلوكي سرّي: الأسلوب، المعاملة، التواصل، التعاون، الاحترافية (١–٥).
- مرة واحدة شهرياً لكل زميل، مع إمكانية إخفاء هوية المُقيِّم.
- ملخص للتقييمات المستلمة بمتوسطات لكل معيار.

### 🏆 التحفيز (Gamification)
- **نقاط** على الإنجاز (تاسك في الموعد / قبل الموعد، تقييم زميل عالي، ستاندب…).
- **أوسمة** (الطائر المبكر، مواظب الستاندب، الأعلى أداءً، روح الفريق).
- **لوحة صدارة** شهرية بترتيب النقاط.

### 💰 البونص
- **هيكل بونص** لكل قسم: بنود بأوزان وحد أقصى وطريقة احتساب.
- **حساب البونص الشهري**: نسبة تحقيق كل بند لكل موظف والسيستم يحسب المستحق،
  بحالات (مسودة / معتمد / مدفوع).

### 🗓️ الأجازات والحضور وسجل الساعات
- **الأجازات:** طلبات (سنوي / مرضي / عارضة) برصيد سنوي وموافقة/رفض من المدير.
- **الحضور:** تسجيل حضور وانصراف، حساب ساعات العمل، وملخص شهري.
- **سجل الساعات (Timesheet):** أسبوعي (السبت → الجمعة بتوقيت مصر) بنسبة
  استغلال + ملخص للفريق.

### 👥 الموظفون والملف الشخصي
- بيانات الموظفين مع **المستوى** (سنيور / جونيور / متدرّب) و**نوع التوظيف**
  (دوام كامل / جزئي / حر) وحالة التفعيل.
- ملف شخصي + روابط السيرة الذاتية (CV) + حقول أمنية.

### 📊 التحليلات ولوحة التحكم
- لوحة تحكم بمؤشرات سريعة + **قسم «محتاج انتباه»** (تاسكات متأخرة، أجازات
  تنتظر مراجعتك، بنود تشيك ليست مفتوحة، بونص لسه مسودة).
- صفحة تحليلات: إجمالي/مكتمل/معدل الإنجاز/المتأخر، وتوزيع التاسكات والمشاريع
  حسب الحالة، مع **رسوم بيانية** و**تصدير CSV**.
- كل الأرقام محترمة لنطاق الدور (شركة كاملة / أقسام القائد / المستخدم نفسه).

### 🧾 سجل التغييرات (Audit)
- **Audit log** بتغييرات النظام، مع عرض مقروء للفروقات (قبل/بعد).

### 🌐 تجربة الاستخدام
- **عربي / إنجليزي** مع تبديل اللغة، واتجاه RTL أصيل.
- **الوضع الداكن** (Dark mode) + تبديل الثيم.
- **PWA**: تثبيت على الموبايل + يعمل بدون اتصال + Service Worker.
- أزرار **طباعة / PDF** و**تصدير CSV**.

## التشغيل محلياً

```bash
# 1. نصب الـ dependencies
npm install

# 2. انسخ ملف البيئة واملأ مفاتيح Supabase
cp .env.example .env.local

# 3. شغّل التطوير
npm run dev
```

افتح [http://localhost:3000](http://localhost:3000). أي صفحة محمية بتحوّل لـ `/login`
طالما مفيش جلسة (مفيش تسجيل عام — الحسابات بينشئها المدير).

## النشر (Production)

1. **Supabase:** أنشئ مشروع، طبّق `supabase/migrations/*.sql` بالترتيب ثم `seed.sql`
   (راجع [`supabase/README.md`](./supabase/README.md)).
2. **الاستضافة:**
   - **Vercel:** اربط الريبو واملأ متغيرات البيئة (راجع `.env.example`).
   - **cPanel / VPS (Node):** راجع [`docs/DEPLOY_CPANEL_AR.md`](./docs/DEPLOY_CPANEL_AR.md)
     (تشغيل عبر `server.js` + الكرون يدوي).
3. **متغيرات البيئة:** مفاتيح Supabase، `SUPABASE_SERVICE_ROLE_KEY`،
   `RESEND_API_KEY`، `DRIVE_TOKEN_ENCRYPTION_KEY`، `GOOGLE_CLIENT_*`،
   `ANTHROPIC_API_KEY` (للذكاء الاصطناعي)، مفاتيح Web Push (`VAPID_*`)،
   `CRON_SECRET`، `NEXT_PUBLIC_APP_URL`.
4. **Cron:** الـ `vercel.json` بيجدول تذكير الستاندب (يومي) والتقييمات الشهرية
   — على استضافة عادية استخدم Cron Jobs تضرب نفس الـ endpoints بالـ `CRON_SECRET`.
5. **DNS:** اضبط SPF/DKIM/DMARC لدومين Resend.

> ملاحظات:
> - الـ Realtime والنسخ الاحتياطي التلقائي محتاجين Supabase Pro.
> - تكامل Google Drive والذكاء الاصطناعي و Web Push **اختيارية** — بتشتغل فقط
>   عند ضبط مفاتيحها، والباقي يفضل شغّال طبيعي.
