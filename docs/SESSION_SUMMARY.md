## 1) إصلاحات حرجة (Bugs & Stability)

### 🔴 إصلاح إنشاء المشروع لا يُحفظ (RLS 403)
- **المشكلة:** الـ server actions كانت تكتب لقاعدة البيانات **بدون توكن المستخدم** فالـ RLS يرفض الكتابة (يعاملها كـ anonymous) — والخطأ كان يُبتلع بصمت.
- **الحل النهائي:** التحقق من الصلاحية في كود التطبيق ثم الكتابة عبر **admin client** (نمط آمن مُوحّد لكل عمليات الكتابة). + إظهار رسائل الخطأ في الواجهة بدل الصمت.

### 🔴 إصلاح شاشة 500 على كل الصفحات
- **المشكلة:** مكوّن `InstallPrompt` كان يستخدم `useTranslations` خارج `NextIntlClientProvider` → خطأ SSR على كل المسارات.
- **الحل:** نقله داخل المزوّد.

### 🟠 الـ Cron يفشل بأمان (Fail-closed)
- نقاط الـ cron كانت مفتوحة للعامة لو `CRON_SECRET` غير مضبوط → الآن ترجع 503 لو غير مضبوط و401 على توكن خاطئ.

## 2) المميزات الرئيسية المضافة

### 💰 حاسبة البونص الشهري
- المدير يسجّل **نسبة تحقيق كل بند (0–100%)** لكل موظف → النظام يحسب `المبلغ = النسبة × الحد الأقصى` والإجمالي.
- حالات: **مسودة → معتمد → مدفوع**، وإشعار للموظف عند الاعتماد/الصرف.
- الموظف يرى بونصه. *(مبني على «هيكل البونص» الموجود مسبقًا.)*
- جدول: `bonus_periods` + `bonus_awards`.

### 🏖️ الأجازات والحضور
- **الأجازات:** طلب أجازة (سنوي/مرضي/عارضة) مع حساب أيام العمل تلقائيًا (يستثني الجمعة/السبت)، رصيد لكل نوع، موافقة/رفض المدير، تحديد الأرصدة السنوية.
- **الحضور:** تسجيل حضور/انصراف يومي + حساب الساعات، وشاشة «حضور الفريق اليوم» + **ملخص ساعات الشهر** لكل موظف للمدير.
- جداول: `leave_balances` + `leave_requests` + `attendance`.

### ⏱️ سجل الساعات والإشغال (Timesheet)
- تسجيل ساعات على التاسكات لأيام الأسبوع، عرض أسبوعي بإجماليات، **نسبة الإشغال** (الساعات ÷ الطاقة) بألوان المناطق، وجدول إشغال الفريق للمدير.
- *بدون جدول جديد — استغلال جدول `time_logs` الموجود.*

### 🤖 مساعدات الذكاء الاصطناعي (Claude)
- **ملخص الستاندب** للمدير، و**Digest أسبوعي** شخصي للموظف.
- تعمل فقط عند ضبط `ANTHROPIC_API_KEY` (تختفي بدونها). الموديل: `claude-opus-4-8`.

### ✅ تشيك ليست المشاريع + قوالب الأقسام
- **تشيك ليست لكل مشروع:** بنود (عنوان + مسؤول + ملاحظات)، المدير يضيف/يعدّل/يحذف، أي عضو يعلّم «تم».
- **قوالب تشيك ليست للقسم:** المدير يعرّف تشيك ليست قياسية → تُطبَّق **تلقائيًا** على أي مشروع جديد + زر «تطبيق القالب» على مشروع موجود.
- جداول: `project_checklist_items` + `checklist_templates`.

### 🔧 أدوات القسم (Tools/Credentials)
- جدول لكل قسم: **اسم الأداة + رابط الدخول + يوزر + باسورد + ملاحظات**.
- الباسورد **مخفي افتراضيًا** مع زر إظهار، والرابط يفتح في تبويب جديد.
- *بدون audit trigger عمدًا حتى لا تُسجَّل الباسوردات في السجل.*
- جدول: `department_tools`.

### 🔔 Web Push (إشعارات الموبايل)
- إشعارات تصل للجهاز **حتى والتطبيق مغلق**، فوق الإشعارات الداخلية والإيميل.
- زر تفعيل/إيقاف لكل جهاز في الملف الشخصي.
- يعمل فقط عند ضبط مفاتيح VAPID. جدول: `push_subscriptions`.

### 👔 دور HR (موارد بشرية)
- دور رابع `hr` يدير شؤون الموظفين **عبر كل الأقسام** (موظفين/أجازات/حضور/بونص/تقييمات/KPIs) — **بدون** سجل التغييرات أو حذف الحسابات أو تعديل بنية الأقسام.
- مبني على دالة `manages_user` في الـ RLS (تحقيق ذكي بدون تعقيد).

### 🆔 بيانات الموظف الإضافية
- **في الملف الشخصي:** رفع **CV** (PDF/Word، bucket خاص بروابط موقّعة) + **رقم واتساب** + **رابط الموقع/البورتفوليو**.
- **تصنيف الموظف:** سنيور / جونيور / متدرّب (يحدده المدير، محمي من تعديل الموظف لنفسه).

### 🌐 واجهة المستخدم
- زر **تبديل اللغة** (عربي ⇄ إنجليزي) وزر **الدارك/لايت مود** في الشريط العلوي (يُحفظان في cookies).
- **داشبورد «محتاج انتباه»:** تجميع للمتأخر والمعلّق والمحتاج موافقة، كل عنصر برابط مباشر.
- **تصدير CSV** لجداول الحضور/الأجازات/الساعات (مع دعم UTF-8 للعربي).
- سطر **«تم التطوير بواسطة Deep Entry | ديب انتري»** تحت فورم تسجيل الدخول (رابط لـ deepentry.net).

## 3) المراجعة الأمنية الشاملة

تمت مراجعة كاملة للكود وقاعدة البيانات:

| الفحص | النتيجة |
|---|---|
| كل الجداول عليها RLS + سياسات | ✅ |
| المستخدم غير المسجّل (anon) | يرى **0 صفوف** من كل الجداول الحساسة |
| الأسرار في الكود | ✅ لا يوجد (كلها `process.env`، ولا تصل للـ client) |
| الـ cron / OAuth / middleware | ✅ مؤمّنة |
| الـ server actions (admin client) | ✅ كل كتابة متحقّق من صلاحيتها — لا CRITICAL/HIGH |

### 🔴 ثغرة خطيرة اكتُشِفت وأُصلِحت — تصعيد صلاحيات
- **المشكلة:** كان بإمكان أي موظف رفع نفسه إلى **super_admin** عبر طلب مباشر لـ Supabase API (سياسة تحديث البروفايل كانت تسمح بتعديل عمود `role`). **أُكِّدت بالاختبار.**
- **الحل:** trigger (`protect_profile_admin_fields`) يحمي الأعمدة الإدارية (`role`, `is_active`, `weekly_hours`, `employment_type`, `hire_date`, `seniority`) — يمنع غير المدير من تغييرها على نفسه. أُكِّد أن التصعيد أصبح ممنوعًا والتعديلات الشخصية تعمل.

## 5) إجراءات يدوية مطلوبة (إعداد)

### في Vercel (Settings → Environment Variables → ثم Redeploy)

```
# Web Push (الإشعارات على الموبايل)
NEXT_PUBLIC_VAPID_PUBLIC_KEY = <المفتاح العام>
VAPID_PRIVATE_KEY            = <المفتاح الخاص — سري>
VAPID_SUBJECT               = mailto:admin@everestads.com

# مساعدات AI
ANTHROPIC_API_KEY = sk-ant-...

# تأكد أنه مضبوط (لتشغيل الـ cron بعد إصلاح الأمان)
CRON_SECRET = <أي نص عشوائي>
```

> توليد مفاتيح VAPID: `node -e "console.log(require('web-push').generateVAPIDKeys())"`

### في لوحة Supabase
- **Authentication → Password:** تفعيل **Leaked Password Protection** (يمنع كلمات السر المسرّبة).

### داخل النظام
- لإنشاء حساب **HR**: المدير العام → الموظفون → دعوة موظف → اختر الدور «موارد بشرية».
- لتفعيل **Web Push**: كل موظف يفتح الملف الشخصي → يفعّل «إشعارات الموبايل» ويقبل الإذن.

## 6) ملاحظات تقنية

- **كل الـ migrations مُطبَّقة بالفعل** على قاعدة بيانات Supabase الحية (لا حاجة لإعادة تشغيلها).
- نمط الكتابة المعتمد: **التحقق من الصلاحية في الكود ثم الكتابة عبر admin client** (لأن JWT المستخدم لا يصل بثبات لـ PostgREST في الـ server actions).
- **تحذير `postcss` (moderate)** في تبعيات `next` — إصلاحه يكسر next؛ يُنتظر تحديث رسمي.
- الباسوردات في «أدوات القسم» مخزّنة كنص (مثل أي vault مشترك)، محمية بـ RLS لأعضاء القسم؛ يمكن إضافة تشفير at-rest لاحقًا كتحسين.
